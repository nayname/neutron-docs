The Revenue module manages validator compensation based on performance metrics and payment schedules.

## Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| `base_compensation` | string | USD value of compensation for validators with 100% performance |
| `denom_compensation` | string | Token denomination used for compensation payments |
| `twap_window_size` | uint64 | Number of blocks for calculating asset price averages |
| `performance_requirements` | object | Performance thresholds for reward eligibility |
| `payment_schedule_type` | enum | Type of payment schedule (monthly, block-based, or empty) |

### Performance Requirements
- `required_at_least`: Minimum performance threshold (0.0-1.0)
- `allowed_to_miss`: Maximum miss rate for full rewards (0.0-1.0)

## Queries

### QueryParams
Retrieve the current module parameters.

**Response:**
- `params`: Current Revenue parameters

### QueryValidatorInfo
Retrieve performance metrics for a specific validator.

**Parameters:**
- `validator_addr`: Validator address to query

**Response:**
- `validator_info`: Performance metrics for the validator

### QueryPaymentSchedule
Retrieve the current payment schedule information.

**Response:**
- `payment_schedule`: Current payment schedule with period tracking

## Messages

### MsgUpdateParams
Update module parameters through governance.

**Authority Required**: Governance only

**Fields:**
- `authority`: Governance authority address
- `params`: New module parameters

### MsgFundTreasury
Fund the treasury for validator compensation.

**Fields:**
- `sender`: Address sending funds
- `amount`: Coins to send to treasury

## Key Concepts

- **Performance-Based**: Compensation based on validator performance metrics
- **Payment Schedules**: Configurable payment periods (monthly or block-based)
- **Price Tracking**: Uses TWAP for asset price calculations
- **Treasury Funding**: Requires funding for validator compensation
- **Query-Only**: Users can only query parameters and validator info
- **Governance Updates**: Parameters can only be updated through governance proposals
