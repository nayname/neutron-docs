---
title: "DEX API"
description: "REST API endpoints for querying DEX module data"
---

The DEX module provides REST API endpoints for querying pool information, reserves, limit orders, and user positions. All endpoints use GET requests and return JSON responses.

## Pool Queries

### Query Pool by ID

```
GET /neutron/dex/pool/{pool_id}
```

Returns information about a specific pool by its ID.

### Query Pool by Parameters

```
GET /neutron/dex/pool/{pair_id}/{tick_index}/{fee}
```

Returns information about a specific pool by its parameters.

**Parameters:**
- `pair_id`: Token pair identifier (e.g., "untrn&lt;&gt;uatom")
- `tick_index`: Tick index of the pool
- `fee`: Fee tier in basis points

### Query Pool Reserves

```
GET /neutron/dex/pool_reserves/{pair_id}/{token_in}/{tick_index}/{fee}
```

Returns reserves for a specific pool.

### Query All Pool Reserves

```
GET /neutron/dex/pool_reserves/{pair_id}/{token_in}
```

Returns reserves for all pools of a token pair.

### Query Pool Metadata

```
GET /neutron/dex/pool_metadata/{id}
```

Returns metadata for a specific pool.

### Query All Pool Metadata

```
GET /neutron/dex/pool_metadata
```

Returns metadata for all pools with pagination support.

## Limit Order Queries

### Query Limit Order Tranche

```
GET /neutron/dex/limit_order_tranche/{pair_id}/{token_in}/{tick_index}/{tranche_key}
```

Returns information about a specific limit order tranche.

### Query All Limit Order Tranches

```
GET /neutron/dex/limit_order_tranche/{pair_id}/{token_in}
```

Returns all limit order tranches for a token pair.

### Query Inactive Limit Order Tranche

```
GET /neutron/dex/inactive_limit_order_tranche/{pair_id}/{token_in}/{tick_index}/{tranche_key}
```

Returns information about an inactive (filled) limit order tranche.

### Query All Inactive Limit Order Tranches

```
GET /neutron/dex/inactive_limit_order_tranche
```

Returns all inactive limit order tranches.

## User Queries

### Query User Deposits

```
GET /neutron/dex/user/deposits/{address}
```

Returns all deposits for a specific user.

**Parameters:**
- `address`: User's address
- `pagination`: Optional pagination parameters
- `include_pool_data`: Whether to include pool data in response

### Query User Limit Orders

```
GET /neutron/dex/user/limit_orders/{address}
```

Returns all limit orders for a specific user.

### Query Limit Order Tranche User

```
GET /neutron/dex/limit_order_tranche_user/{address}/{tranche_key}
```

Returns user's position in a specific limit order tranche.

### Query All Limit Order Tranche Users

```
GET /neutron/dex/limit_order_tranche_user
```

Returns all limit order tranche user positions.

## Liquidity Queries

### Query Tick Liquidity

```
GET /neutron/dex/tick_liquidity/{pair_id}/{token_in}
```

Returns tick liquidity for a token pair.

## Simulation Endpoints

### Simulate Deposit

```
GET /neutron/dex/simulate_deposit
```

Simulates a deposit transaction. Requires MsgDeposit parameters.

### Simulate Withdrawal

```
GET /neutron/dex/simulate_withdrawal
```

Simulates a withdrawal transaction. Requires MsgWithdrawal parameters.

### Simulate Place Limit Order

```
GET /neutron/dex/simulate_place_limit_order
```

Simulates placing a limit order. Requires MsgPlaceLimitOrder parameters.

### Simulate Multi-Hop Swap

```
GET /neutron/dex/simulate_multi_hop_swap
```

Simulates a multi-hop swap transaction. Requires MsgMultiHopSwap parameters.

### Simulate Withdraw Filled Limit Order

```
GET /neutron/dex/simulate_withdraw_filled_limit_order
```

Simulates withdrawing from a filled limit order. Requires MsgWithdrawFilledLimitOrder parameters.

### Simulate Cancel Limit Order

```
GET /neutron/dex/simulate_cancel_limit_order
```

Simulates canceling a limit order. Requires MsgCancelLimitOrder parameters.

## Module Parameters

### Query Parameters

```
GET /neutron/dex/params
```

Returns the module parameters.

## Pagination

List endpoints support standard Cosmos SDK pagination parameters:
- `pagination.key`: Next page key
- `pagination.offset`: Number of items to skip
- `pagination.limit`: Maximum items per page
- `pagination.count_total`: Whether to count total items
- `pagination.reverse`: Whether to reverse order

## Response Format

All endpoints return JSON responses. The exact response structure depends on the specific endpoint and matches the corresponding protobuf message definitions.

For detailed response schemas, refer to the protocol buffer definitions in the module source code.
