## Governance Messages

The Cron module accepts messages through governance proposals.

### AddSchedule

Creates a new schedule through governance proposal.

```protobuf
message AddSchedule {
  string name = 1;                           // Unique identifier for the schedule
  uint64 period = 2;                         // Number of blocks between executions
  repeated MsgExecuteContract msgs = 3;      // Messages to execute
  string execution_stage = 4;                // Execution stage
}
```

### RemoveSchedule

Removes an existing schedule through governance proposal.

```protobuf
message RemoveSchedule {
  string name = 1;                           // Name of the schedule to remove
}
```

## Query Methods

### Params

Returns the module parameters.

**CLI Usage:**
```bash
neutrond query cron params
```

### Schedule

Returns a specific schedule by name.

**CLI Usage:**
```bash
neutrond query cron schedule [schedule-name]
```

### Schedules

Returns all schedules.

**CLI Usage:**
```bash
neutrond query cron schedules
```

## Module Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `security_address` | string | Address authorized to remove schedules |
| `limit` | uint64 | Maximum number of schedules executed in one block |

## Events

The module emits events for schedule operations:

- `EventAddSchedule` - When a schedule is created
- `EventRemoveSchedule` - When a schedule is removed  
- `EventExecuteSchedule` - When a schedule is executed
